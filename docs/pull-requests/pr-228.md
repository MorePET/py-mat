## `pymat.materials` registry

Single object, three usage shapes — mirrors `pint.UnitRegistry`'s call/attr/subscript trifecta:

```python
pymat.materials                              # Mapping[str, Material]
pymat.materials["Stainless Steel 304"]      # subscript (mypy-clean)
pymat.materials("Stainless Steel 304")      # callable lookup
pymat.materials()                            # all materials → list
pymat.materials(category="metals")          # filter → list
pymat.materials(grade="316L")
pymat.materials(tags=["austenitic"])         # multi-axial
pymat.materials(with_vis=True)
"X" in pymat.materials                      # membership
for name, m in pymat.materials.items(): ...  # full Mapping API
len(pymat.materials)
```

Drops the earlier `pymat.lookup()` proposal (redundant with the callable form). Keeps `pymat["X"]` shorthand for 3.4-3.10 BC.

Mixed positional + filter kwargs raise `TypeError` (lookup-with-filter is ambiguous) — caught both by static `@overload` and runtime guard.

## Tags schema (#132)

New `Material.tags: list[str]` field. Multi-axial labels orthogonal to the TOML hierarchy (chemistry / function / industry / treatment / regulation). TOML field `tags = [...]`; loader merges parent chain into effective tag list (children **extend** parent's tags, don't replace).

Backfilled stainless TOML with canonical example tags:
- `[stainless]` → `["ferrous", "stainless", "corrosion-resistant"]`
- `[stainless.s316L]` → child extends with `["austenitic", "marine-grade", "molybdenum-bearing", "316-family"]`

(Wider tag backfill across all 157 materials in a follow-up data PR — #132 phase 2.)

## `search()` filter extensions

`pymat.search()` accepts the same domain filter kwargs as `materials()`: `category`, `grade`, `tags`, `with_vis`. Filters narrow the candidate set BEFORE the fuzzy/exact match runs.

Visual filters (`source`, `tier`, PBR scalars) stay in `pymat.vis.search()` where they live — different namespace, different concern.

Empty query returns `[]` regardless of filters (preserves 3.4-3.10 contract; pure filter/browse goes through `pymat.materials()` instead).
