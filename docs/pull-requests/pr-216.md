# [PR 216](https://github.com/MorePET/mat/pull/216) docs(readme): add 3.11 feature examples to compiled README

## Summary

The README is generated from `tests/test_readme_examples.py` via `scripts/generate_readme.py`, with a pre-commit hook (`readme-from-tests`) locking the two in sync. The 3.11 surface ‚Äî `_sources` provenance, `_stddev` sugar, composition ranges, T-dependent property curves, and the new Phase 5 catalog entries ‚Äî wasn't covered by any of those tests, so:

1. **The README didn't mention any of the 3.11 features**, even though they're the headline of the release.
2. **Nothing was guarding against drift** ‚Äî code changes to the new APIs wouldn't break a doc test.

This PR closes that gap by adding four test classes (each method = one runnable docstring example):

- `TestProvenance` ‚Äî `Material.source_of(path)`, `Material.cite(path)`. Pins BGO's Weber & Monchamp 1973 DOI and Inconel 625's MIL-HDBK-5J page citation.
- `TestUncertainty` ‚Äî `<prop>_stddev` sibling sugar (from a `tmp_path` TOML) and `aluminum.a6063` composition with `{nominal, min, max}` windows. Asserts the ufloat round-trip and the `density_g_mm3` build123d-boundary coercion.
- `TestTemperatureCurves` ‚Äî `<prop>_curve` + `<prop>_at(T)` linear interpolation, exercised against an OFHC-copper-style knot set (77 K / 295 K / 500 K).
- `TestCuratedCatalog` ‚Äî smoke tests for GAGG:Ce, CeBr3, SrI2:Eu, Inconel 625, Inconel 718. Asserts headline scalars so README examples can't drift from the curated corpus silently.

README regenerated; `python scripts/generate_readme.py --check` exits 0.
